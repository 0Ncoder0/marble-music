# Feature Specification: Music Physics Sandbox

**Feature Branch**: `001-music-physics-sandbox`  
**Created**: 2026-04-17  
**Status**: Draft  
**Feature ID**: `001-music-physics-sandbox`

---

## User Scenarios & Testing *(mandatory)*

### User Story 1 — 搭建场景并触发碰撞音乐 (Priority: P1)

玩家进入网页后，在画布上放置小球、方块和音乐方块，搭建一个物理结构。按下空格键启动播放，小球在重力驱动下滚动弹跳，每次碰到音乐方块就发出一个音符声音。玩家无需任何音乐背景，即可在 10 分钟内完成"搭建 → 发声"的完整体验。

**Why this priority**: 这是产品的最核心交互闭环。没有这条路径，任何其他功能都失去意义。

**Independent Test**: 在一个空场景中放置 1 个小球、1 个斜坡方块、1 个音乐方块 → 按空格键 → 验证小球下落并在碰撞音乐方块时产生可闻声音，即完整验证此故事。

**Acceptance Scenarios**:

1. **Given** 空场景（编辑模式），**When** 玩家放置一个小球和一个音乐方块后按空格，**Then** 进入播放模式，小球受重力下落，碰撞音乐方块时立即发出钢琴音色，碰撞后声音随时间自然衰减至静音。
2. **Given** 播放模式正在运行，**When** 玩家再次按空格或按 Esc，**Then** 物理模拟停止，场景恢复编辑模式，右侧面板和底部 Timeline 重新显示。
3. **Given** 播放模式，**When** 玩家尝试点击画布放置新积木，**Then** 操作被完全忽略，场景中没有新实体出现。

### User Story 2 — 编辑态实时预览碰撞结果 (Priority: P2)

玩家在编辑模式下搭建结构时，画布上出现虚线预测轨迹，底部同步显示五线谱（Timeline），预先告知哪个小球在哪个时刻会触发哪个音符。玩家可通过观察五线谱来判断布局是否符合创作意图，无需反复进入播放模式验证。

**Why this priority**: 实时预测是区别本产品与普通物理沙盒的核心创作价值。它让玩家在"看到"音乐之后再"听到"音乐，大幅降低试错成本。

**Independent Test**: 放置 1 个小球和 1 个音乐方块 → 验证画布上出现预测轨迹虚线，底部 Timeline 五线谱出现该小球对应的音符；然后移动音乐方块位置，验证五线谱中音符位置（时间轴）随之更新。

**Acceptance Scenarios**:

1. **Given** 编辑模式，场景中有至少 1 个小球和 1 个音乐方块，**When** 场景稳定，**Then** 画布上显示该小球的预测轨迹虚线，底部 Timeline 五线谱显示对应预测音符（时间 × 音高位置）。
2. **Given** 编辑模式，**When** 玩家新增、移动或删除任意实体，**Then** 预测轨迹和 Timeline 五线谱同步刷新，反映变更后的预测结果。
3. **Given** 编辑模式，**When** 玩家修改某音乐方块的音名，**Then** Timeline 五线谱中该方块对应音符的音高位置更新。
4. **Given** 玩家按空格进入播放模式，**Then** 预测轨迹虚线消失，底部 Timeline 五线谱整体隐藏；**When** 按空格或 Esc 返回编辑模式，**Then** 两者重新显示，显示内容基于当前场景重新计算。
5. **Given** 多个小球场景，**Then** 每个小球在 Timeline 五线谱中拥有独立一行谱线，各自显示该球的预测碰撞音符，以小球标识（颜色/编号）区分。

### User Story 3 — 配置音乐方块音乐参数 (Priority: P2)

玩家在编辑模式下点击某个音乐方块，右侧面板显示该方块的音名和音量参数。玩家通过选择器调整音名（决定音高），通过滑块调整音量（影响发声强度和持续时长）。修改后预测系统立即同步更新。

**Why this priority**: 音乐参数决定了"音乐性"。没有参数调整，玩家只能创作出单调的声音，无法进行有意义的音乐创作。

**Independent Test**: 选中一个音乐方块 → 将音名从 C4 改为 G4 → 验证 Timeline 五线谱中该音符纵向位置改变；将音量从 0.5 改为 0.1 → 播放后验证声音明显更短促。

**Acceptance Scenarios**:

1. **Given** 编辑模式，**When** 玩家点击一个音乐方块，**Then** 右侧面板显示该方块的当前音名和音量值。
2. **Given** 音乐方块已选中，**When** 玩家将音名改为有效音名（如 "A3"），**Then** 参数立即生效，Timeline 五线谱同步更新，音符位置反映新音高。
3. **Given** 音乐方块已选中，**When** 玩家将音量滑块从 0.8 调低到 0.2，**Then** 进入播放模式后，该方块触发的音符声音更短促；高音量时声音更绵长。
4. **Given** 参数面板可见，**When** 玩家输入非法音名（如空字符串或格式错误），**Then** 界面即时给出红色提示，音名不被写入，旧值保留。
5. **Given** 播放模式，**Then** 右侧参数面板不可见，任何参数修改途径均被阻断。

### User Story 4 — 多球场景与相机跟随 (Priority: P3)

玩家在编辑模式下放置多个小球，点击某个球将其选中。按空格进入播放模式时，若有选中球则相机自动平滑跟随该球运动；若没有选中球则相机保持当前位置。播放期间玩家始终可手动平移和缩放视窗。

**Why this priority**: 多球场景是中级创作能力的体现。跟随策略确保玩家在复杂场景下不会"迷失"，同时保留手动控制自由度。

**Independent Test**: 放置 2 个小球 → 点击选中球 A → 按空格播放 → 验证相机追踪球 A 的运动；停止 → 不选中任何球 → 按空格播放 → 验证相机不自动移动。

**Acceptance Scenarios**:

1. **Given** 场景中有 2 个以上小球，**When** 玩家点击某个小球，**Then** 该球被选中（视觉高亮），且同一时刻只有一个球可被选中。
2. **Given** 有选中球，**When** 玩家按空格进入播放模式，**Then** 相机以平滑插值方式跟随该球移动，而不是瞬间跳跃。
3. **Given** 无选中球，**When** 玩家按空格进入播放模式，**Then** 相机保持当前视窗位置，不自动移动。
4. **Given** 播放模式且相机正在跟随，**When** 玩家滚动鼠标滚轮或使用中键拖拽，**Then** 相机响应手动操作。
5. **Given** 播放模式停止返回编辑，**Then** 上一次的选中球状态保留，直到玩家显式选中另一球。

### User Story 5 — 自动保存与场景恢复 (Priority: P3)

玩家在编辑模式下的所有操作（放置、移动、删除、参数修改）自动保存到浏览器本地存储。刷新页面后，场景结构和音乐参数完整恢复，无需手动保存操作。保存成功或失败状态通过右上角 HUD 显示。

**Why this priority**: 自动保存是基础可信度保障。玩家会在一次 session 中反复调整，任何数据丢失都会严重损害信任感。

**Independent Test**: 放置 3 个音乐方块并调整各自音名 → 刷新页面 → 验证 3 个方块的位置和音名与刷新前完全一致，Timeline 五线谱显示相同的预测结果。

**Acceptance Scenarios**:

1. **Given** 编辑模式下玩家进行了任意操作，**When** 操作完成约 1~2 秒后，**Then** 当前场景自动保存到浏览器本地存储，右上角 HUD 短暂显示"已保存"。
2. **Given** 已保存的场景，**When** 玩家刷新页面，**Then** 所有实体（小球、方块、音乐方块）的位置、尺寸、旋转角度完整恢复，音乐方块的音名和音量参数完整恢复。
3. **Given** 本地存储不可用或配额已满，**When** 保存操作失败，**Then** 右上角 HUD 持续显示"保存失败"以及可行动提示（如"请检查浏览器存储权限"），不静默丢失。
4. **Given** 浏览器本地存储数据损坏，**When** 玩家刷新页面，**Then** 显示空场景并给出"存档损坏"提示，不崩溃。
5. **Given** 玩家从播放模式返回编辑模式，**Then** 立即触发一次强制保存，不等待节流窗口。

### Edge Cases

- **空场景启动播放**：场景中没有任何实体时，按空格仍可进入播放模式，物理引擎空转，无发声，无崩溃。
- **音乐方块无小球碰撞**：Timeline 五线谱显示空谱线（无音符），播放时也无发声，行为正常。
- **极高频碰撞**：小球卡在两个方块之间高速弹跳时，音频系统启用声音实例上限保护，不产生爆音或浏览器崩溃，跳过超限触发但已有声音正常继续。
- **浏览器音频未授权**：页面加载后首次用户交互前尝试播放，系统提示用户"点击以启用音频"；物理模拟可继续（静音模式）。
- **连续快速切换模式**：快速多次按空格键，系统保持稳定，不出现 UI 状态混乱或音频错误。
- **大量实体场景**：场景包含 20 个积木和 5 个小球时，预测计算和渲染保持可用性，无明显卡顿。
- **删除被选中的球**：在编辑模式删除当前选中球后，`selectedBallId` 清空，下次按空格播放时相机不跟随。
- **恢复后音频首次触发**：刷新恢复场景后，首次按空格播放时音频上下文需要用户交互才能激活，系统正确处理此场景。

## Requirements *(mandatory)*

### Functional Requirements

#### 实体管理

- **FR-001**: 系统必须支持玩家在编辑模式下通过点击画布空白区域放置三种实体：小球（Ball）、方块（Block）、音乐方块（Music Block）。
- **FR-002**: 系统必须支持玩家通过右侧积木选择器（或快捷键 `1`/`2`/`3`）选择当前放置的实体类型。
- **FR-003**: 玩家必须能够在编辑模式下通过点击实体将其选中，并通过拖拽将选中实体移动到新位置。
- **FR-004**: 玩家必须能够在编辑模式下通过 `Delete` 或 `Backspace` 键删除当前选中的实体。
- **FR-005**: 系统必须在编辑模式下对每个实体显示选中高亮效果，同一时刻最多只有一个实体处于选中状态。

#### 模式系统

- **FR-006**: 系统必须在任意时刻处于且仅处于两种模式之一：编辑模式或播放模式，不存在暂停或其他中间状态。
- **FR-007**: 按下 `Space` 键时，若处于编辑模式则切换到播放模式；若处于播放模式则停止并返回编辑模式。
- **FR-008**: 在播放模式下按下 `Esc` 键，必须停止播放并返回编辑模式。
- **FR-009**: 在编辑模式下按下 `Esc` 键，必须取消当前临时放置态（如已选择积木类型但尚未放置），不切换模式。
- **FR-010**: 进入播放模式时，系统必须锁定一切编辑操作（新增、移动、删除实体，修改参数），并隐藏右侧编辑面板和底部 Timeline 五线谱。
- **FR-011**: 返回编辑模式时，系统必须恢复编辑操作权限，重新显示右侧面板和底部 Timeline 五线谱，并触发一次预测重算。

#### 物理与发声

- **FR-012**: 在播放模式下，物理引擎必须驱动小球（Ball）受重力影响运动；方块（Block）和音乐方块（Music Block）始终保持静止。
- **FR-013**: 每当小球与音乐方块发生物理碰撞时，系统必须立即触发该音乐方块对应的钢琴音符发声（碰撞到发声延迟低于 20ms）。
- **FR-014**: 每次碰撞触发必须创建独立的音符实例，允许多个音符同时重叠发声，不使用冷却抑制机制。
- **FR-015**: 每个音符发声后必须按音量自然衰减至静音，无显式音长参数；音量越高衰减时间越长，音量越低衰减时间越短。
- **FR-016**: v1 固定使用钢琴（piano）音色，不提供音色选择功能。

#### 音乐方块参数

- **FR-017**: 玩家必须能够在编辑模式下选中音乐方块后，在右侧参数面板中修改其音名（noteName，如 "C4"、"A3"）和音量（volume，范围 0~1）。
- **FR-018**: 音名和音量的修改必须实时生效，无需确认操作；非法输入（空音名、超范围音量）必须即时给出视觉提示并拒绝写入。
- **FR-019**: 音量参数同时决定发声时的初始响度和自然衰减时长，在参数面板中应有明确说明。

#### 预测系统

- **FR-020**: 在编辑模式下，系统必须在画布上以虚线形式显示每个小球的预测运动轨迹（预测线）。
- **FR-021**: 在编辑模式下，系统必须在底部 Timeline 区域以五线谱形式显示预测碰撞音符序列；多个小球时每个球独占一行谱线，以小球标识区分。
- **FR-022**: 场景中任何影响物理结果的变更（实体新增/移动/删除、参数修改）必须触发预测线和 Timeline 五线谱的同步重算和刷新。
- **FR-023**: 预测线和 Timeline 五线谱必须基于同一次预测模拟结果，保证空间轨迹与音符序列完全一致。
- **FR-024**: 进入播放模式时，预测线和 Timeline 五线谱必须隐藏且停止计算；返回编辑模式时必须重新显示并触发重算。

#### 相机与视窗

- **FR-025**: 玩家在编辑和播放两种模式下均可通过鼠标滚轮缩放视窗、通过中键拖拽或 `Alt+左键拖拽` 平移视窗。
- **FR-026**: 在编辑模式下，玩家点击小球可将其设为"当前选中球"（selectedBall），同一时刻最多一个球被选中。
- **FR-027**: 按空格进入播放模式的瞬间，若存在选中球，则相机以平滑插值方式自动跟随该球；若无选中球，则相机保持当前位置不自动跟随。
- **FR-028**: 播放期间相机跟随可被玩家手动平移/缩放操作覆盖；停止播放返回编辑后，选中球状态保留。

#### 持久化

- **FR-029**: 编辑模式下的所有场景变更（实体操作、参数修改）必须在操作后约 1~2 秒内自动保存到浏览器 localStorage。
- **FR-030**: 从播放模式返回编辑模式时，必须立即触发一次强制保存，不受节流延迟约束。
- **FR-031**: 页面加载时必须自动读取并恢复上次保存的场景，包括所有实体位置、尺寸、旋转角度及音乐方块的音名和音量参数。
- **FR-032**: 恢复场景成功后，系统必须立即触发一次预测计算，使 Timeline 五线谱显示恢复后场景的预测结果。
- **FR-033**: 存档不存在时，系统必须创建空场景；存档损坏（JSON 解析失败、数据不完整）时，系统必须回退到空场景并给出可感知提示，不崩溃。
- **FR-034**: 保存成功时右上角 HUD 短暂显示"已保存"；保存失败时持续显示"保存失败"并附可行动提示，不静默丢失。

#### 视觉与反馈

- **FR-035**: 音乐方块发声时，必须在该方块位置显示脉冲/扩散动画（如扩散环），动画亮度和大小与音量同步，随声音衰减同步消退。
- **FR-036**: 当前模式（编辑/播放）必须在界面中有明确的视觉提示（如左上角 HUD），确保玩家不会误判当前状态。

### Key Entities *(include if feature involves data)*

- **Ball（小球）**: 动态物理实体，在播放模式下受重力运动。关键属性：位置（x, y）、速度（vx, vy）、半径（radius）。编辑态速度始终为零，可被选为跟随目标。
- **Block（方块）**: 静态碰撞体，构成轨道、斜坡、边界结构。关键属性：位置（x, y）、宽高（width, height）、旋转角度（rotation）。始终不移动。
- **Music Block（音乐方块）**: 静态碰撞体，被小球碰撞时触发音符。关键属性：位置（x, y）、宽高、音名（noteName）、音量（volume），音色固定为 piano。无音长参数（durationMs 不存在）。
- **Scene（场景）**: 顶层容器，持有所有实体及选中球 ID。当前模式在存档中始终保存为编辑态。重力参数可配置。
- **PredictedNote（预测音符）**: 纯运行时数据，不持久化。记录某次预测模拟中，某小球在某时刻碰撞某音乐方块的预测结果（时间、音名、音量），用于渲染 Timeline 五线谱。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 玩家可在首次进入的 10 分钟内独立完成一次"放置实体 → 查看 Timeline 五线谱预测 → 按空格播放 → 听到碰撞发声 → 返回编辑调整"的完整闭环，全程无需任何说明文档。
- **SC-002**: 在标准复杂度场景（不超过 20 个积木、不超过 5 个小球）下，编辑、预测、播放整体体验流畅，视觉上无明显卡顿感。
- **SC-003**: 碰撞发声的视听同步感良好，玩家主观感知"碰到即响"，无明显延迟感（目标低于 20ms 碰撞到发声延迟）。
- **SC-004**: 播放模式下对编辑操作的阻断率为 100%——任何新增、移动、删除、参数修改尝试均不产生效果，场景不被意外改变。
- **SC-005**: 刷新页面后，场景实体和音乐参数的恢复完整率为 100%——所有实体的位置、尺寸、旋转和音乐方块参数与刷新前完全一致。
- **SC-006**: 保存失败时，玩家 100% 能感知到明确提示（非静默失败），并获得可行动建议。
- **SC-007**: Timeline 五线谱的预测结果与实际播放的碰撞音序在整体音符顺序上一致（允许极小的时间误差），玩家主观认为"预测与结果吻合"。
- **SC-008**: 多实例重叠发声功能稳定可用：场景中多个音乐方块在短时间内被连续碰撞时，各音符均能独立发声并自然衰减，无相互干扰或截断。

## Assumptions

- **平台限制**：v1 仅面向 PC Web 环境（桌面浏览器 Chrome / Edge / Firefox），不支持移动端或触屏设备。
- **音色固定**：v1 固定使用钢琴音色，不提供音色选择功能；音色扩展推迟到后续版本。
- **无导入导出**：v1 不支持场景文件的导入或导出，不支持 MusicXML 或任何外部乐谱格式。
- **无账号与云存储**：场景数据仅保存在浏览器本地 localStorage，不涉及网络请求、账号系统或云端同步。
- **无暂停功能**：交互模型仅有编辑和播放两态，不提供暂停。设计决策已确认。
- **无音长参数**：音乐方块参数只有音名（noteName）和音量（volume），不提供显式音长（durationMs）设置；音量驱动的自然衰减已是设计定稿。
- **Timeline 纯预测**：Timeline 五线谱是编辑态的预测工具，播放态完全隐藏且不计算。它反映的是预测结果，不是播放状态的实时同步。
- **存档键固定**：localStorage 使用固定的键名保存场景，不支持多存档槽位。
- **重力可配置但默认固定**：重力参数在数据模型中存在（默认向下），v1 不提供面向玩家的重力调整 UI。
- **选中状态跨会话不保留**：selectedBallId 虽然进入持久化（支持跟随恢复），但其核心用途是播放启动时的跟随判定；刷新后若恢复该值，行为与刷新前保持一致。
